## Step 8: Select One Semester Project Opportunity

### Selected Opportunity

**WatchMate – Movie Recommendation System**

### Review of NUF Scores

The Movie Recommendation System received a high NUF score because it is useful to many users, reasonably innovative, and feasible to develop within one semester.

### Reality Check

The idea is realistic because:

* Many users struggle to find movies that match their interests.
* Movie datasets are publicly available.
* Basic recommendation features can be implemented without advanced AI.
* The project scope can be adjusted to fit the semester timeline.

### Validation for Lab 03

The team can validate the problem by:

* Surveying university students and movie viewers.
* Interviewing potential users about how they currently discover movies.
* Measuring how much time users spend searching for movies.

### Prototype Feasibility (Lab 05 – Lab 10)

A prototype can be developed during the semester with features such as:

* User preference selection.
* Movie search and filtering.
* Personalized movie recommendations.
* Movie details page.
* Watchlist feature (optional).

### Tools and Technology

* Figma for UI/UX design.
* HTML, CSS, and JavaScript for frontend development.
* Movie dataset or movie API for recommendation data.
* Simple database or local storage for user preferences.

### Final Team Decision

After reviewing the NUF scores and discussing feasibility, the team selected **WatchMate – Movie Recommendation System** as the semester project because it addresses a real user problem, can be validated in Lab 03, and is achievable as a working prototype by Labs 05–10.
