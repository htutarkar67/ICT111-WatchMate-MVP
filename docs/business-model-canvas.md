# Business Model Canvas

## Project Title
WatchMate: AI Movie Recommendation System

---

## 1. Customer Segments

- Movie enthusiasts looking for personalized movie recommendations.
- University students and busy individuals who want to find movies quickly.
- Casual viewers who are unsure what to watch.
- Administrators who monitor chatbot usage and user interactions.

---

## 2. Value Proposition

WatchMate provides an AI-powered movie recommendation platform that helps users discover movies and TV shows through natural conversations. Instead of spending a long time browsing streaming platforms or searching online, users simply describe their preferences, mood, or favorite movies, and the AI chatbot generates personalized recommendations. The system saves users time, improves the movie selection experience, and makes discovering new content easier.

---

## 3. Channels

- Web application accessible through desktop and mobile browsers.
- University project demonstrations and presentations.
- GitHub repository for project documentation.
- Social media and promotional materials for future product awareness.

---

## 4. Customer Relationships

- Friendly and interactive AI chatbot for personalized recommendations.
- Easy-to-use interface for searching and browsing movies.
- Personalized chat history for logged-in users.
- User feedback to improve the recommendation experience.
- Administrator monitoring of chatbot conversations to enhance service quality.

---

## 5. Revenue Streams / Sustainability Logic

This is a non-commercial university project developed for educational purposes. The system demonstrates how artificial intelligence can improve movie discovery and recommendation experiences.

In the future, the platform could generate revenue through:
- Premium subscription for advanced AI recommendation features.
- Affiliate partnerships with streaming platforms.
- Advertisements for movies and entertainment services.
- Partnerships with cinema chains or movie distributors.

---

## 6. Key Resources

- Development team.
- AI chatbot API (e.g., Google Gemini/OpenAI).
- TMDB API for movie information.
- Web technologies (HTML, CSS, JavaScript, React, Node.js).
- Database for storing user accounts and chat history.
- GitHub for version control and collaboration.

---

## 7. Key Activities

- Develop and maintain the web application.
- Integrate the AI chatbot and movie database.
- Provide personalized movie recommendations.
- Maintain movie information and chatbot functionality.
- Test the system and improve user experience.
- Collect user feedback for future improvements.

---

## 8. Key Partners

- TMDB (The Movie Database) API provider.
- AI service provider (Google Gemini/OpenAI).
- Project supervisors and instructors.
- Development team members.
- Users who provide feedback to improve recommendations.

---

## 9. Cost Structure

- Development time.
- AI API usage costs.
- TMDB API integration and maintenance.
- Web hosting and deployment.
- Database hosting.
- Software development tools.
- System maintenance and future updates.

---

## Final Note

The business model supports the MVP by focusing on the essential features required for an AI-powered movie recommendation system, including AI chatbot recommendations, movie search, trending movies, movie detail pages, chat history, watchlist functionality, and administrator monitoring of chatbot conversations. These core features address the problem of information overload by helping users discover movies more quickly and efficiently while providing a simple and engaging user experience.
